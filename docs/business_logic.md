# AIPower-Efficiency-Pilot 业务逻辑说明书 (池化定价版)

## 1. 业务逻辑全景图
本项目旨在构建 **“资源池感知 -> 池化计费 -> 跨池分析 -> 闭环治理”** 的全生命周期效能闭环。

## 2. 阶段一：池化感知 (Pool-Based Trace)
*   **核心逻辑：**
    1.  **扫描池子元数据：** 通过解析 Pod 调度标签，识别所在的资源池名称（如 `Infer-A100-MIG-Pool`）。
    2.  **提取切分模式 (Slicing Mode)：** 自动从池子名称中提取 `Full/MIG/MPS/TS` 标识，并映射至对应的驱动层特性。
    3.  **多维绑定：** 建立 `Pod <-> Pool <-> Slicing_Mode <-> Physical_GPU` 的四层关联。

## 3. 阶段二：池化差异化计费 (Pool Billing)
*   **计费公式：** `费用 = Pool_Base_Price * Slicing_Weight * Duration`。
*   **各模式计费细则：**
    *   **Full (独享):** 按照整卡物理成本计费，侧重“资源占位”收益。
    *   **MIG (硬切分):** 按照硬件切片 Profile (1g, 2g, 3g...) 的权重比例分摊成本。
    *   **MPS (软并行):** 支持算力分摊，结合“显存申请量”与“算力活跃度”动态计费。
    *   **TS (时间片/超分):** 应用低价策略，单价受池子整体“超分密度”动态调节（密度越高，单价越低）。

## 4. 阶段三：跨池效能画像 (Analyze)
*   **核心任务：** 识别任务与资源池的不匹配，给出优化建议。
*   **分析对象：** 任务（Pod/PyTorchJob），而非资源池。
*   **任务画像：** 自动识别任务类型（训练/推理/开发测试）、优先级（高/中/低）、硬件需求（高端特性）。
*   **治理建议类型：**
    *   **降配**：减少资源用量，不换池
    *   **迁移**：换到更合适的资源池
    *   **降配+迁移**：既减资源又换池，节省更多
*   **智能决策：**
    *   重要任务（高优先级/有特殊硬件需求）：保守处理，只降配不迁移
    *   普通任务：可以迁移到更低成本资源池
    *   每条建议给出 1-2 个选项，让用户选择
*   **输出：** AI 生成的《任务治理建议报告》，包含问题描述、建议选项、预估效果。
*   **产出：** AI 生成的《任务治理专家报告》，包含 1-2 条建议选项。

## 5. 阶段四：治理闭环 (Governance)
*   **核心目标：** 将 AI 诊断建议转化为实际治理动作，真正实现成本节省。
*   **治理来源：** 已批准的 AI 诊断报告。
*   **治理动作：**
    *   **降配**：减少 GPU 数量，不换池
    *   **迁移**：换到更合适的资源池
    *   **降配+迁移**：既减 GPU 又换池
*   **执行方式：** 用户选择任务 → 预览变更内容 → 确认执行 → 系统自动完成治理
*   **状态追踪：** 执行状态实时更新

> 注意：本项目为 FinOps 平台，专注于治理建议和执行能力。实际利用率改善和费用节省需要运行后持续监控数据，不在本项目范围内。
