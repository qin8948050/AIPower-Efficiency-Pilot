### ASCII 架构流转说明：
1. **集群层源头：** Pod 被调度在不同的池子中，消耗异构物理资源。
2. **感知层捕获：** K8s 事件流与 Prometheus 时间序列双重汇聚，确保不因监控延迟导致账单丢失。
3. **数据层缝合：** 将“这几秒钟的事件”和“当时的硬件指标”在 MySQL 缝缝补补成为财务实体快照。
4. **大脑层诊断：** 一边生成真金白银的数字账单，一遍通过大模型降维分析输出“省钱大招”。
5. **执行层闭环：** AI给出建议 -> 转换为 K8s 配置代码 -> 人为查看安全确认 -> 回到集群执行降维打击。
